1 · How Studio lives inside BrandrAI

One app, two meters, three product lines.

Studio is not a separate product or login. It is a top-level workspace section inside app.brandrai.com that sits alongside COA Verify and (later) Proof and Sites. The user has one account, one subscription, one billing relationship вҖ” but two independent meters that refill on the 1st of every month:

COA verifications

Authentication product. One verification = one COA run through the pipeline (OCR вҶ’ rules вҶ’ hash). Overage: $2/verify.

Studio credits

Generation product. Variable cost per action (1-20cr). Overage: buy packs (Taste $35 / Creator $129 / Studio $449 / Agency $1899).

The two meters never bleed into each other. You can never "spend" a COA verification on a Studio generation, and Studio credits cannot be used for COA verification. This is intentional: COA is an authentication product with regulatory weight (each verify is a signed assertion); Studio is a creative tool where cost varies by action.

Rules the UI must follow:

  • Studio screens show the Studio credit meter only. COA Verify screens show the COA verification meter only. Never both side-by-side in content area (it causes "which one got charged?" confusion).
  • Both meters are always available in the sidebar bottom panel (section 2).
  • Before any generation action: show вңЁ 4 credits on the CTA button вҖ” e.g. [Generate image · 4cr].
  • After action: toast confirmation вҖ” "Image generated. 4 credits used. 1,838 remaining."
  • When < 10% remaining: meter turns amber, sidebar shows "Buy more" CTA.
  • When 0 remaining: generation CTAs change to [Buy credits to generate] (not greyed out вҖ” actively sells).

8 · Upsell moments + pack purchase

Four moments to sell a credit pack. Don't miss them.

Moment 1 · < 10% remaining

Sidebar meter turns amber. Soft banner on Studio home: "You're almost out for the month. Grab a pack to keep going вҖ” Creator pack is $129 for 500 credits (same rate as your plan)."

Moment 2 · Zero remaining

Generate CTAs swap to [Buy credits to generate]. Clicking opens pack picker modal. No dead-end — the button sells.

Moment 3 · Expensive action > remaining

User clicks Generate video В· 20cr but only has 8cr left. Inline modal: "Not enough credits for this action. Add 100 credits for $35 to continue?" вҖ” one-click Taste pack purchase.

Moment 4 · Month-end high usage

If user finished the month > 80% used for 2 consecutive months — suggest tier upgrade in billing panel, not pack (sticky ARR beats one-time pack).
